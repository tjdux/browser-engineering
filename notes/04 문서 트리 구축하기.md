## 01 노드 트리

- DOM 트리: 태그 쌍을 위한 노드(node)와 텍스트 스펜(span)을 위한 노드를 가지고 있음
- 토큰을 노드로 변경: 자식 노드들의 리스트와 부모 노드에 대한 포인터 필요
- 파싱 (parsing)
  - 소스 코드로부터 노드 트리를 구성하는 것
  - parser: 한 번에 하나의 엘리먼트나 텍스트 노드를 추가하면서 트리를 구축
  - 🏹 파서가 파싱을 진행하는 동안 불완전한 트리를 저장해야 함
  - 미완성 태그: 항상 열려 있지만 아직 닫히지 않았고, 완성된 노드보다 소스에서 항상 뒤에 나오고, 항상 다른 미완성 태그의 자식 노드
    <br/>

## 02 파서 디버깅하기

- `__repr__` 함수
  - 개발자 입장에서 명확하게 식별할 수 있는 문자열로 표현하기 위한 특수 메서드
  - `repr(obj)`를 호출하면 내부적으로 `obj.__repr__()`이 실행
  - 디버깅, 로깅, 개발 콘솔에서 객체를 확인할 때 사용
  - 일반적으로 모든 데이터 객체에 대해 `__repr__` 메서드를 정의하고 해당 `__repr__` 메서드가 모든 관련 필드를 인쇄하도록 하는 것이 좋음
    <br/>

## 06 페이지 오류 다루기

- 암시적 태그 (implicit tag)

```html
<!DOCTYPE html>
<html>
  <head></head>
  <body></body>
</html>
```

- `<!DOCTYPE html>`를 제외한 여섯 태그는 모두 선택 사항
- 웹페이지가 해당 태그들을 빼먹었다면 브라우저들이 그 태그들을 자동으로 삽입

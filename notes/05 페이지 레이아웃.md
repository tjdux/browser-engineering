## 01 레이아웃 트리

- 브라우저에서의 레이아웃: 레이아웃 트리를 만드는 것
- 레이아웃 트리의 노드들: HTML 엘리먼트와 연결되어 있고 각각 크기와 위치를 갖는 레이아웃 오브젝트가 됨
- 브라우저는 DOM 트리를 순회하면서 레이아웃 트리를 만들고, 각 레이아웃 객체의 크기와 위치를 계산한 후, 각각의 레이아웃 객체들이 화면에 그려지게 됨
  <br/>

## 02 블록 레이아웃

- 웹페이지는 헤딩, 문단, 메뉴와 같은 큰 블록들로 구성되어 있으며, 이 블록들이 수직으로 쌓여있음
- 루트 노드 `DocumentLayout` + `BlockLayout` 객체로 구성된 전체 트리를 갖기
  <br/>

## 03 크기와 위치

- x 위치
  - 부모 블록의 x 위치가 이미 계산되어 있어야 함
  - ➡️ 부모 블록의 x는 자식 블록들의 x보다 먼저 계산되어야 함
  - ➡️ x를 계산하는 작업이 재귀적인 `layout` 호출 이전에 수행되어야 함
- height
  - 자식의 높이에 따라 height 값 달라짐
  - ➡️ height는 재귀적인 `layout` 호출 후에 계산해야 함
- y 위치
  - 이전 형제 노드의 y 위치에 따라 달라짐
  - ➡️ 재귀적인 `layout` 호출은 첫 번째 형제 노드에서 시작하여 리스트를 순방향으로 이터레이션
- 🚀 `BlockLayout`의 `layout` 메서드의 높이, 너비 계산 단계 (종속성 추론)
  - 1️⃣ 먼저 `parent`와 `previous` 레이아웃 객체로부터 읽은 값으로 `width`, `x`, `y`필드 계산
  - 2️⃣ 각 자식 엘리멘트를 위한 자식 레이아웃 객체 생성
  - 3️⃣ 각 개체의 `layout` 메서드를 호출하여 자식 레이아웃 노드를 재귀적으로 배치
  - 4️⃣ 마지막으로 `layout`은 자식 레이아웃 객체에서 읽은 값으로 `height` 계산

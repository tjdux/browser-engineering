## 01 폰트(서체)란?

- 폰트: 글꼴, 서체, 타입
- 폰트: 글자 두께 (e.g. bold, normal), 스타일과 크기 (e.g. italic)
- Tk의 폰트객체: 고정된 크기, 스타일 및 글자 두께의 정보를 담고 있음

```python
import tkinter.Font

window = tkinter.Tk()
bi_times = tkinter.Font.Font(
  family="Times",
  size=16,
  weight="bold",
  slant="italic",
)

# 폰트 객체를 create_text에 font 인수로 전달
canvas.create_text(200, 100, text="Hi!", font=bi_times)
```

## 02 텍스트 측정하기

- 폰트 객체의 `metrics()`, `measure()`: 텍스트가 차지하는 가로세로 공간 측정

### `metrics()`

```python
bi_times.metrics()
# {'ascent': 15, 'descent': 4, 'linespace': 19, 'fixed': 0}
```

- `metrics()`: 텍스트의 세로축에 대한 정보들을 제공
- `ascent`: 기준선 윗부분의 높이
- `descent`: 기준선 아랫부분의 높이
- `linespace`: 텍스트의 높이
  - 폰트 크기를 16으로 설정했는데 19픽셀이라고 보고하는 이유
    - 폰트 크기가 16 == 72 분의 1인치를 의미하는 16 포인트 (16픽셀이 아님!)
    - 실제로 폰트 크기가 16인 폰트들의 글자들은 높이가 다양함
    ```python
    tkinter.Font.Font(family="Courier", size=16).metrics()
    # {'fixed': 1, 'ascent': 13, 'descent': 4, 'linespace': 17}
    tkinter.Font.Font(family="Times", size=16).metrics()
    # {'fixed': 0, 'ascent': 14, 'descent': 4, 'linespace': 18}
    tkinter.Font.Font(family="Helvetica", size=16).metrics()
    # {'fixed': 0, 'ascent': 15, 'descent': 4, 'linespace': 19}
    ```
- `fixed`: 부울값으로 모든 문자의 너비가 같은지를 알려줌
- `ascent`, `descent`
  - 크기가 다른 글자가 같이 있을 때 필요
  - 크기가 다른 글자가 같은 줄에 있을 때는 위나 아래가 아닌 기준선으로 정렬해야 함

### `measures`

- 텍스트가 차지하는 가로 공간의 크기를 알려줌
- 글자마다 너비가 다르기 때문에 같은 폰트라도 텍스트에 따라 크기가 다름

```python
bi_times.measure("Hi!")
// 31
```

### 텍스트 측정하기

- `Hello, world!` 텍스트를 두 조각으로 나누어 `world!`는 이탤릭채로 그린다고 가정

```python
# 폰트 준비
font1 = tkinter.Font.Font(family="Times", size=16)
font2 = tkinter.Font.Font(family="Times", size=16, slant="italic")

# (200, 200)에서 시작하여 텍스트 그리기
x, y = 200, 200 # 텍스트의 중심 위치
canvas.create_text(x, y, text="Hello, ", font=font1, anchor="nw") #anchor="nw": 좌표를 텍스트 좌상단 모서리로 처리
x += font1.measure("Hello, ")
canvas.create_text(x, y, text="world!", font=font2)
```

## 06 다양한 크기의 텍스트

- 영어: 모든 글자가 보이지 않는 기준선 (baseline)에 정렬된 상태로 작성됨
- `<small>a</small><big>A</big>`를 제대로 출력하기 위해서는...
  - 작은 텍스트 (`<small>a</small>`)를 아래로 이동시켜야 함 (큰 텍스트(`<big>A</big>`)를 위로 이동시키면 위의 줄과 겹침)
  - 작은 텍스트가 먼저 `token`함수를 통과하기 때문에 텍스트 줄에 대한 2패스 알고리즘 (두 번 계산)이 필요 - 첫 번째 패스: 줄에 어떤 단어가 들어가는지 식별하는 것과 x 위치를 계산 - 두 번째 패스: 단어를 수직으로 정렬하고 y 위치를 계산
